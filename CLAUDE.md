# HCAD Appeal Tool

A free property tax appeal tool for Harris County, TX homeowners. Users enter their property account number or address, the tool finds comparable properties from HCAD public data, calculates if they're overassessed, and generates a ready-to-file appeal packet (PDF + HTML).

## Project Architecture

```
hcad-appeal/
├── frontend/          # React + Vite SPA
├── backend/           # Express.js API + SQLite
│   ├── routes/        # API route handlers
│   ├── scripts/       # ETL and data scripts
│   └── db/            # SQLite DB and schema
└── data/              # HCAD raw CSV files (gitignored, large)
```

## Tech Stack

- **Frontend:** React 18, Vite, TailwindCSS
- **Backend:** Express.js, better-sqlite3
- **PDF:** pdf-lib (programmatic), printable HTML fallback
- **Auth:** Not implemented yet — but structure all user-facing data with an optional `user_id` FK so auth can be layered in later

## Data

HCAD publishes annual bulk data exports at https://hcad.org/hcad-resources/hcad-appraisal-data/
Key files used:
- `Real_acct_owner.txt` — account/owner records
- `building_res.txt` — residential building details (sqft, beds, baths, year built)
- `land.txt` — land data

ETL script: `backend/scripts/import-hcad.js`
- Downloads or reads from `data/` directory
- Parses pipe-delimited or CSV files
- Loads into SQLite tables: `properties`, `buildings`, `owners`
- Run once per year when HCAD releases new data

## Key Features

1. **Property Lookup** — search by account number or street address
2. **Comparable Finder** — find comps by zip code, similar sqft (±20%), year built (±10yr), same property class
3. **Overassessment Score** — calculate % above/below median assessed value per sqft for comps
4. **Appeal Packet Generator** — produce:
   - Cover page (property summary + potential savings)
   - Formal appeal letter (cites Texas Property Tax Code §41.41)
   - Comparable sales evidence table
   - Filing instructions (Harris County ARB process)
   - Deadline checklist
   - Legal disclaimer
5. **PDF Export** — `pdf-lib` generated, downloadable
6. **Print View** — styled HTML for browser print

## API Endpoints

```
GET  /api/property/:accountNumber     — fetch property by HCAD account number
GET  /api/property/search?q=          — search by address
GET  /api/comps/:accountNumber        — find comparables
POST /api/appeal/generate             — generate appeal packet data
GET  /api/appeal/:id/pdf              — download PDF
GET  /api/appeal/:id/html             — view printable HTML
```

## Database Schema

```sql
-- Core property data from HCAD bulk export
properties (account_number PK, address, city, zip, land_value, improvement_value, total_value, tax_year)
buildings (id, account_number FK, sqft, year_built, beds, baths, stories, condition, quality)
owners (id, account_number FK, owner_name, mailing_address)

-- Generated by the app
appeal_packets (id, account_number, created_at, user_id nullable, packet_json, status)
```

## Comparable Selection Logic

1. Same zip code (relax to adjacent zips if < 5 comps found)
2. Sqft within ±20% of subject property
3. Year built within ±10 years
4. Same property class (residential)
5. Sort by assessed value per sqft
6. Return top 10 comps; flag if subject is in top 25% (likely overassessed)

## Coding Guidelines

- Keep backend routes thin — logic in service modules
- All DB queries use prepared statements (never string interpolation — SQL injection risk)
- Frontend components: one responsibility each, no god components
- No auth yet, but add `user_id` nullable column to `appeal_packets` from the start
- PDF generation is server-side only (pdf-lib runs in Node)
- Error states must be user-friendly — these are non-technical homeowners
- Deadlines are year-specific (May 15 in TX) — don't hardcode, derive from `tax_year`

## Running Locally

```bash
# Backend
cd backend && npm install
node scripts/import-hcad.js   # one-time data load
npm run dev                   # starts on :3001

# Frontend
cd frontend && npm install
npm run dev                   # starts on :5173
```
